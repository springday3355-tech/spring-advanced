# SPRING ADVANCED

- 이번 프로젝트에서는 기존 코드의 성능 최적화, 가독성 향상 그리고 테스트코드의 정합성을 맞추는 리펙토링 작업을 수행했습니다


## 1. 코드 개선

1) Early Return 패턴 적용을 통한 성능 최적화
- 문제: 회원가입 시에 비용이 큰 비밀번호 암호화 로직이 이메일 중복체크 보다 먼저 실행되는 상태였음
- 해결: 이메일 중복 체크를 최상단으로 옮겨, 중복 발생시 바로 예외를 던지도록 수정함
- 효과: 불필요한 암호화 연산을 뒤로 옮겨 서버 자원을 낭비하지 않게함

2) 불필요한 else 블록 제거
- 문제: 중첩된 if-else 구조로 인해 가독성이 떨어졌었음
- 해결: 예외상황 먼저 처리하는 Guard Clauses 방식을 사용해 else 없이 간결하게 구현함
- 효과: 코드 들여쓰기 단계가 줄어들어 로직 파악이 쉬워졌음

3) Spring Validation 라이브러리 도입
- 문제: 서비스 계층에서 if-else를 사용해 수동으로 데이터 유효성을 검증함
- 해결: DTO에 @NotBlank, @Size, @Pattern 어노테이션 적용 후 컨트롤러에서 @Valid로 검증함
- 효과: 비즈니스 로직과 검증 로직을 DTO로 분리하여 코드의 유지보수성을 높임

4) N+1 문제 해결(@EntityGraph)
- 문제: 연관된 엔티티를 조회할 때 데이터 개수 만큼 추가 쿼리가 발생하는 N+1 현상이 발견되었음
- 해결: JPQL의 Fetch Join 대신에 스프링 데이터 JPA의 @EntityGraph를 사용해 연관데이터를 한 번에 조회하도록 변경함
- 효과: DB 시스템 부하를 줄이고 데이터 속도를 최적화할 수 있음

## 2. 테스트 코드 연습

[테스트 케이스 정합성 수정] 
1) PasswordEncoder 테스트: matches() 메서드 호출 시 인자 순서 즉, 원본 비밀번호와 암호화된 비밀번호의 순서를 바꿔 해결함
2) 예외처리 테스트: 특정 상황에서 발생하는 예외가 NPE이 아닌 의도한 예외인 InvalidRequestException인지 확인 후 테스트 메서드 명을 직관적으로 변경함 
3) CommentService 테스트:
- 문제: 댓글 등록 실패테스트에서 기대한 클래스가 ServerException으로 잘못표기되어 테스트 실패하였음 
- 해결: 실제 서비스 로직에서 던지는 InvalidRequestException으로 기대 예외 클래스를 수정하여 정상화함
4) ManagerService 로직 수정
- 문제: 일정에 작성자가 없는 경우에 대한 검증로직이 없어서 테스트 실패함
- 해결: ManagerService의 saveManager 로직에 if (todo.getUser() == null) 검증문을 추가해 일정을 생성한 유저가 없을 때 예외 메세지를 던지도록 수정
- 효과: 예외 상황에 대한 방어로직 강화하고 테스트코드와 비즈니스로직의 일관성을 확보하였음

느낀점: 실행에 문제없는데 어떤 에러를 고치고 수정해야하는지 난감했지만, 리펙토링 하는 것을 통해 서버의 성능향상과 코드의 가독성이
중요하다는 것을 배울 수 있었고, 테스트 코드가 실패할 때 단순히 테스트 코드만 고치는 것이 아니라, 실제 서비스 로직의 빈틈을 찾아 보완해야 한다는 점을 깨닫게 되었다
